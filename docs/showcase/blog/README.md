## 写作约定（3 篇共用）

1. **每篇开头必有"反直觉钩子"** — 不能用"今天我们聊聊 X"这种平庸开头
2. **每篇必有可验证数据** — 不能纯讲故事，要带具体数字（4/4、10/10、24h、$400→$80、+457%）
3. **每篇必有 1 张原创架构图** — 复用 [diagrams/](../diagrams/) 已有的 Mermaid 源码
4. **每篇结尾必有"行动召唤"** — GitHub repo / 简历 / 订阅 / 评论
5. **避免营销腔** — 不写"赋能"、"颠覆"、"全栈式解决方案"
6. **平实表达** — 让没做过 Agent 的高级工程师也能读懂
