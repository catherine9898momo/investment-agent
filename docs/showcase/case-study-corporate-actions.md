# Case Study：LLM 训练知识不稳定 — 24 小时设计并落地 ground truth 兜底架构

> investment-agent 项目反例闭环复盘 · 2026-05-13 ~ 2026-05-14
> 用途：简历附件 / 邮件附带 / 终面前置阅读材料
> 阅读时长：3 分钟 · 转 PDF：A4 一页

## 一句话摘要

**3 个 MCP Server 协作的金融 Agent 跑通后**，发现 LLM 训练知识在不同 session 输出不一致（同一持仓两次给出 **15:1 vs 3:1** 不同拆股精度，复权后成本相差 5 倍）。**24 小时内**设计并落地 `corporate_actions_server` ground truth 兜底架构，抽象出"**触发器 / 具体事实 / 决策建议**"三层知识分层设计模式与 **LLM hop 最小化**原则。

---

## S — Situation（背景）

`investment-agent` 在 W2 阶段（2026-05-13）跑通 3 个 MCP Server 协作（memory / finance / news），跨 Server 拆股事件判别 4/4（NVDA / TSLA / AAPL / 09988.HK）。当时叙事："Agent 内化金融领域常识，主动识别拆股导致的成本基准错误"。

## T — Task（问题暴露）

**5/13 当天晚上，同一持仓两次 session 给出不一致答案**：

| 时间 | 问句 | 拆股识别 | 复权计算 | 浮盈方向 |
|---|---|---|---|---|
| **早上** | "我的 TSLA 浮盈多少" | 2020-08 (5:1) + 2022-08 (3:1) **累计 15:1** | $1200 ÷ 15 = **$80** | **正确** |
| **晚上** | "持仓 TSLA + 新闻 + 建议" | 仅识别 "2022 年 1:3" | $1200 ÷ 3 = **$400** | **方向反转** |

历史事实：TSLA 真的有两次拆股，**早上对、晚上漏算 2020-08**。问题严重性：

1. 用户无法判断哪个答案对（两次都"权威感"输出）
2. 决策影响巨大（浮盈 vs 浮亏方向反转）
3. Agent 没有"我不确定"信号

**这个反例直接证伪了昨天"LLM 训练知识可白嫖"的乐观结论**。

## A — Action（行动方案）

### 归因（机制层）

不是"训练数据过时"，是 LLM **注意力机制天然存在的压缩/忽略偏移**：长 session + 多工具调用 + 输出长建议的上下文压力下，关键事实被概括化处理。同一模型同一天可以输出不同精度。

### 设计（架构层）

**知识分层设计模式**：

| 层 | 角色 | 提供者 | 容错率 |
|---|---|---|---|
| 触发器 | "可能有问题"信号 | LLM 训练知识 | 高（容损） |
| **具体事实** | **日期 / 比例 / 数字** | **外部 ground truth** | **必须 0** |
| 决策建议 | 持有 / 加仓 / 减仓 | LLM 综合 | 高（容损） |

核心原则：**LLM 提示要存在感、不要权威感**。LLM 应该说"可能是拆股前价位，建议核对"，不应该直接说"$80 USD，浮盈 +361%"。

### 实施（工程层，5/14 上午一次性完成）

- `corporate_actions_server.py`：yfinance 拉权威数据 + SQLite 24h TTL 缓存 + stale fallback
- 接口：`get_corporate_actions(symbol) → {actions, cumulative_split_factor, source, last_fetched}`
- 在 `memory.add_holding` / `finance.get_quote` 的 description 里 nudge："涉及历史价位判断时先调本工具拿 ground truth"
- 注册到 Claude Desktop config（`investment-corporate-actions`）

## R — Result（结果，5/14 当日实测验证）

**Case D' 真实复现 4/4 维度全过**：

| 维度 | 反例（5/13 晚） | 修复后（5/14） |
|---|---|---|
| 工具调用次数 | 4 | **7**（自动调用 get_corporate_actions） |
| 拆股识别 | 仅 2022-08 1:3 | **2020-08 5:1 + 2022-08 3:1 累计 15:1** |
| 复权后成本 | $400 | **$80** |
| 浮盈方向 | -64% 浮亏 | **+457% 浮盈** |

**3 个未设计的涌现行为**（额外加分）：

1. **金句逐字命中**：Claude 自发输出"价位异常 → 先查历史拆股事实"——与设计文档里"知识分层"措辞高度一致
2. **主动澄清持仓时点**：弹 4 选项让用户确认买入时间，体现"存在感 vs 权威感"的 UX 落地
3. **跨 Server 协同提议**：主动建议更新 portfolio 口径（没人教过的协同行为）

**全程闭环时长 24 小时**：5/13 晚发现 → 5/14 上午实施 → 5/14 当日实测验证。

## 元教训（工程原则升华）

1. **LLM hop 最小化**：每经过一次 LLM 解析 = 一次注意力损失风险。关键事实应该 hop=0（结构化工具直给字段），模糊判断可以 hop>0（LLM 越想越好）。
2. **垂直 Agent 的本质分水岭**：通用 Agent 把数据扔给用户；垂直 Agent 结合领域常识做判断 + 给修正方案。但"领域常识"不能直接从 LLM 训练知识取——必须有 ground truth 兜底。
3. **反例叙事 > 顺利叙事**：识别系统局限 + 主动修复 = P6+ 主导能力证据。比"看，我的 Agent 多聪明"强一个档次。

## 简历叙事升级

**反例闭环前**：
> Agent 内化金融领域常识，主动识别拆股可能导致的成本基准错误。

**反例闭环后**：
> Agent 在 4 个拆股 case 上展现金融常识介入（4/4 判别），**但同时发现 LLM 训练知识在长 session 下的不稳定性**——同一持仓在不同 session 给出 15:1 vs 3:1 不同复权计算。**24 小时内**设计并实施 `corporate_actions_server` 作为外部 ground truth 来源，让 LLM 训练知识只做"触发器"，权威数字由结构化工具提供——这是垂直领域 Agent 在生产环境下必须的"**知识分层**"设计模式，背后的机制层归因是 **LLM 注意力机制的压缩偏移**。
