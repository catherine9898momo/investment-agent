1. **CLI 会话入口**
   - 启动：
     ```bash
     python3 -m src.agents.research_demo
     ```
   - 单轮：
     ```bash
     python3 -m src.agents.research_demo --query "美光最近为什么会大跌？"
     ```
   - 支持：
     ```text
     /help
     /config
     /json
     /debug
     /exit
     ```

2. **live-first**
   - 默认 `--data-source live`
   - `fixture` 只作为测试模式：
     ```bash
     python3 -m src.agents.research_demo --data-source fixture --query "美光最近为什么会大跌？"
     ```
   - live 失败不会崩，也不会伪装成真实数据，会生成缺失事实。

3. **用户问题理解**
   - 能识别：
     - 美光 / Micron / MU
     - 为什么跌 / 大跌 / 下跌 / 回调
     - 估值
     - 持仓
     - 直接交易建议

4. **时间窗口解析**
   - “最近为什么大跌”会解析成最近约一周 / 5 个交易日。

5. **归因计划**
   对“为什么跌”会规划：
   ```text
   price_move
   news_events
   corporate_actions
   sector_move
   peer_moves
   macro_context
   earnings_or_guidance
   analyst_actions
   ```

6. **Verified/Missing Fact**
   - 已有工具事实进入 `verified_facts`
   - 未覆盖归因需求进入 `missing_facts`
   - 报告会显示：
     ```text
     已形成 x 条可追踪事实，另有 x 项证据缺口
     ```

7. **用户报告不再暴露工程 ID**
   默认报告不显示：
   ```text
   fact_xxx
   src_xxx
   claim_xxx
   guardrail checks
   evidence table
   trace path
   ```

8. **研究结论和原因排序**
   有足够价格/新闻线索时，会生成：
   ```text
   研究结论
   原因排序
   ```
   但目前原因排序还是规则关键词版本，不是完整 LLM synthesis。
