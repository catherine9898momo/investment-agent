可以，而且这正是 investment agent 从“能回答问题”升级成“研究工作台”的关键：**投资问题不是无限发散的，它可以收敛成一组常见研究意图，每类意图都有稳定 workflow，可以沉淀成 planner template。**

我会先分成这些方向。

| 投资问题方向 | 用户常见问法 | 常规处理模式 |
|---|---|---|
| **单公司研究** | “帮我看 TSLA 是否值得继续关注” | 公司识别 → 行情/新闻/财务/公司行动 → thesis/risk → evidence memo |
| **行业研究** | “AI 服务器行业有没有前景？” | 行业定义 → 需求/供给/产业链/竞争格局 → 公司池 → 风险与反证 |
| **估值分析** | “现在贵不贵？” | 估值指标 → 历史分位 → 同业对比 → 增长/利润率假设 → 安全边际 |
| **财报解读** | “这季财报怎么样？” | 财报事实抽取 → 同比/环比 → guidance → 市场预期差 → thesis 是否变化 |
| **新闻/事件归因** | “为什么今天跌？” | 价格变化 → 新闻事件 → 市场/行业/同行对比 → 是否能归因 → 不确定性 |
| **组合风险** | “我现在组合风险大吗？” | 持仓读取 → 行业/地区/因子暴露 → 集中度 → 回撤/相关性 → 风险 memo |
| **买卖边界问题** | “现在能买/要不要卖？” | 识别交易建议请求 → 拒绝直接指令 → 转成研究框架/确认问题 |
| **监控/提醒** | “帮我盯着 MU 的风险” | thesis/invalidator → 监控指标 → 触发规则 → mini memo |
| **投资原则匹配** | “这家公司符合巴菲特/芒格标准吗？” | 原则 checklist → 公司事实 → 匹配/不匹配 → 缺失证据 |
| **宏观/周期问题** | “降息对科技股影响？” | 宏观变量 → 传导链条 → 受益/受损资产 → 历史类比 → 风险 |
| **对比研究** | “NVDA 和 AMD 哪个更好？” | 标的标准化 → 指标并排 → 护城河/增长/估值/风险 → 差异总结 |
| **投资 thesis 管理** | “我之前看好它的逻辑还成立吗？” | 读取 thesis → 新事实对照 → strengthened/weakened/unchanged → invalidators |

这些方向都可以沉淀成模式。形式上不是写死答案，而是沉淀成 **Research Workflow Template**：

```text
Intent Type
 -> Required Facts
 -> Optional Facts
 -> Tools / Data Providers
 -> Normalizers
 -> Memo Sections
 -> Guardrail Rules
 -> Missing Fact Policy
 -> Regression Cases
```

举个例子，**行业研究 template** 可以是：

```text
Intent: industry_research

Required Facts:
- industry_definition
- demand_drivers
- supply_constraints
- competitive_structure
- value_chain_profit_pool
- representative_companies
- valuation_context
- key_risks
- thesis_invalidators

Workflow:
1. 识别行业边界和地区
2. 拆分产业链
3. 收集需求侧事实
4. 收集供给侧事实
5. 找利润池和代表公司
6. 做估值/周期/风险分析
7. 输出行业 memo
8. 标出 missing facts 和 human confirmation points
```

**财报解读 template** 可以是：

```text
Intent: earnings_review

Required Facts:
- reported_revenue / EPS / margin / FCF
- YoY / QoQ change
- guidance
- consensus expectation if available
- management commentary
- stock reaction
- thesis impact

Output:
- 本季发生了什么
- 超预期/低预期在哪里
- 哪些是一次性因素
- 哪些改变长期 thesis
- 还需要确认什么
```

**组合风险 template** 可以是：

```text
Intent: portfolio_risk_review

Required Facts:
- holdings
- position weights
- sector/geography exposure
- correlation / beta / volatility
- max drawdown or stress scenario
- single-name concentration
- liquidity / currency risk

Output:
- 最大风险来源
- 集中度风险
- 情景压力测试
- 缺失数据
- 非交易建议的人工确认问题
```

我建议你在项目里沉淀成一个新的概念：**Intent Playbook**。

它可以长这样：

```python
IntentPlaybook(
    intent="industry_research",
    required_fact_needs=[...],
    optional_fact_needs=[...],
    default_tools=[...],
    memo_sections=[...],
    guardrail_rules=[...],
    missing_fact_policy="degrade_with_unknowns_or_retrieve",
)
```

然后 orchestrator 的 planner 就不是每次临场发挥，而是：

```text
User Query
 -> Intent Router
 -> Select Intent Playbook
 -> Instantiate Research Plan
 -> Execute / Retrieve / Normalize
 -> Source / Fact
 -> Synthesize / Verify / Memo
```

这非常值得沉淀，因为它解决三个问题：

1. **可复用**：行业、公司、财报、估值、组合风险都有稳定套路。
2. **可评估**：每个 intent 可以有自己的 regression cases。
3. **可解释**：面试时可以说清楚 agent 为什么这么查，而不是“LLM 自己觉得该这么查”。

对当前 investment-agent 来说，下一步最自然就是加一层：

```text
QueryIntake
 -> IntentRoute
 -> IntentPlaybook
 -> ResearchPlan
```

目前你已经有 `QueryIntake`、`IntentRoute`、`ResearchPlan` 的雏形。差的是把不同投资问题类型正式沉淀成 playbook/template。这个方向很对，而且很有产品化味道。
